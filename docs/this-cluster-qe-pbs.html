이 자료는 숙제용 읽기 slide deck입니다. 목표는 “어디에서 파일을 준비하고, 어디에서 job을 제출하고, 어디에서 결과를 찾는지”를 초보자 기준으로 익히는 것입니다.

1. 서버 구조

로그인 노드와 계산 노드를 구분합니다.

2. 작업 위치

home과 scratch 역할을 구분합니다.

3. 제출 방법

PBS script를 만들고 qsub로 제출합니다.

Before you startprompt labels and safety

자료에 나오는 server$는 “누리온에서 입력”이라는 표시입니다.

맞는 입력

server$ pwd

실제로는 pwd만 입력합니다.

틀린 입력

server$ pwd

server$까지 복사하면 명령어가 아닙니다.

중요: 로그인 노드에서 큰 QE 계산을 직접 실행하지 않습니다. 계산은 PBS script를 만들어 qsub로 제출합니다.

누리온은 내 노트북이 아니라, 많은 계산 노드가 모여 있는 슈퍼컴퓨터입니다.

내 노트북

Windows / macOS

login01

계산 노드

여러분이 직접 들어가는 곳은 로그인 노드입니다. 계산 노드는 PBS scheduler가 배정합니다.

server$ hostname server$ whoami server$ pwd server$ echo $HOME

현재 폴더 위치를 확인합니다. 초보자는 자주 확인하는 습관이 중요합니다.

Storage modelhome vs scratch

누리온에서는 home과 scratch를 다르게 사용합니다.

위치역할초보자 기억법
/home01/$USER입력 파일, PBS script, 중요한 결과를 보관합니다.정리해서 오래 보관할 작업 공간.
/scratch/$USER실제 계산을 실행하고 큰 output/tmp 파일이 생기는 공간입니다.계산을 돌리는 작업 공간. 영구 보관 장소가 아닙니다.
이 클러스터에서는 /home01에서 qsub를 하면 실패할 수 있습니다. job 제출은 /scratch/$USER 아래에서 합니다.
Workflowprepare · copy · submit · backup

기억할 핵심 문장: home에서 준비하고, scratch에서 실행하고, 결과는 다시 home에 보관합니다.

1. home 준비

~/qe_projects/qe_test

입력 파일과 PBS script 작성.

›

2. scratch 복사

/scratch/$USER/...

계산 실행 폴더로 복사.

›

3. qsub 제출

qsub run_qe.pbs

반드시 scratch에서 제출.

›

4. 결과 백업

~/qe_projects/.../results

중요 파일은 home으로 복사.

Important warningscratch is temporary

/scratch에만 중요한 결과를 두지 마세요.

server$ ls /scratch/$USER # 오래 접근하지 않은 파일은 삭제 대상이 될 수 있습니다.
누리온 문서에 따르면 scratch 파일은 일정 기간 접근하지 않으면 삭제 대상이 될 수 있습니다. 삭제 대상 파일에는 ToBeDelete_ 같은 접두사가 붙을 수 있습니다.

초보자용 결론

  • 계산 중에는 scratch를 사용합니다.
  • 계산이 끝나면 중요한 파일을 home으로 복사합니다.
  • 큰 tmp/ 폴더는 필요한지 담당자에게 확인합니다.
Step 1prepare files in home

먼저 home에 원본 계산 폴더를 만듭니다.

server$ mkdir -p ~/qe_projects/qe_test server$ cd ~/qe_projects/qe_test server$ pwd

여기에 둘 파일

  • scf.in — QE 입력 파일
  • run_qe.pbs — PBS 제출 script
  • pseudo/ — pseudopotential 폴더

home은 정리된 원본을 두는 장소라고 생각하면 됩니다.

Step 2copy files to scratch

계산을 실행할 scratch 폴더를 만들고 파일을 복사합니다.

server$ mkdir -p /scratch/$USER/qe_projects/qe_test server$ cp -r ~/qe_projects/qe_test/* /scratch/$USER/qe_projects/qe_test/ server$ ls -lh /scratch/$USER/qe_projects/qe_test

cp

copy, 파일을 복사합니다.

-r

폴더 안까지 recursive하게 복사합니다.

*

현재 폴더 안의 여러 파일을 뜻합니다. 사용 전 위치를 확인하세요.

Small file skillscheck before submitting

제출 전에 파일 목록과 이름을 꼭 확인합니다.

이름 바꾸기

mv는 move의 줄임말이지만, 같은 폴더 안에서 쓰면 “이름 바꾸기”가 됩니다.

server$ mv old_name.in new_name.in
server$ cd /scratch/$USER/qe_projects/qe_test server$ pwd /scratch/내ID/qe_projects/qe_test server$ ls -lh # scf.in, run_qe.pbs, pseudo/ 가 보이나요?
초보자 습관: qsub 전에 항상 pwd와 ls를 실행합니다.
Step 3submit a PBS job

job은 반드시 /scratch 아래에서 제출합니다.

server$ cd /scratch/$USER/qe_projects/qe_test server$ pwd /scratch/내ID/qe_projects/qe_test server$ qsub run_qe.pbs

정상 위치

pwd 결과가 /scratch로 시작합니다.

자주 보는 에러

qsub: ERROR: You can't submit job in home01 directory, please try in scratch directory

Step 4monitor or cancel jobs

제출 후에는 PBS 명령으로 job 상태를 확인합니다.

server$ qstat -u $USER server$ qstat -f JOBID server$ qdel JOBID
server$ qstat -q server$ showq server$ pbs_status server$ pbs_queue_check server$ isam

qstat -u

내 job 목록을 봅니다.

qdel

job을 취소합니다. JOBID를 정확히 넣습니다.

isam

SRU time 사용량을 확인합니다.

Step 5backup results to home

계산이 끝나면 중요한 결과를 home에 보관합니다.

server$ mkdir -p ~/qe_projects/qe_test/results server$ cp -r /scratch/$USER/qe_projects/qe_test/* ~/qe_projects/qe_test/results/ server$ ls -lh ~/qe_projects/qe_test/results
큰 계산에서는 tmp/ 폴더가 매우 클 수 있습니다. restart 계산이 필요하면 tmp/가 필요할 수 있으므로, 처음에는 담당자에게 “tmp도 복사해야 하나요?”라고 확인하세요.

QE modulesarchitecture matters

QE module을 load할 때 queue/architecture에 맞는 module을 사용합니다.

debug 테스트: KNL

module purge module load craype-mic-knl module load intel/oneapi_21.2 module load impi/oneapi_21.2 module load qe/7.5

norm_skl 계산: SKL

module purge module load craype-x86-skylake module load intel/oneapi_21.2 module load impi/oneapi_21.2 module load qe/7.5
초보자용 구분: debug 테스트는 craype-mic-knl, norm_skl 계산은 craype-x86-skylake를 사용합니다.
PBS scriptdirectives at the top

PBS script 맨 위에는 scheduler에게 줄 요청사항을 씁니다.

#!/bin/bash -l #PBS -N qe_test #PBS -A qe #PBS -q debug #PBS -l select=1:ncpus=2:mpiprocs=2 #PBS -l walltime=00:05:00 #PBS -j oe
A

#PBS -A qe

QE job script에는 이 줄이 필요합니다.

Q

#PBS -q debug

queue를 지정합니다. 테스트는 짧게 시작합니다.

T

walltime

최대 실행 시간을 지정합니다.

PBS scriptbody that runs QE

script 본문에서는 작업 폴더로 이동하고, module을 load하고, MPI로 QE를 실행합니다.

cd "$PBS_O_WORKDIR" module purge module load craype-mic-knl module load intel/oneapi_21.2 module load impi/oneapi_21.2 module load qe/7.5 NP=$(wc -l < "$PBS_NODEFILE") echo "host: $(hostname)" echo "workdir: $(pwd)" echo "PBS_JOBID: $PBS_JOBID" echo "NP: $NP" which mpirun which pw.x which xspectra.x mpirun -np "$NP" pw.x -in pw_dummy.in > pw_dummy.out 2>&1 mpirun -np "$NP" xspectra.x -in xspectra_dummy.in > xspectra_dummy.out 2>&1
Launch testsmall before real

가장 작은 launch test부터 제출합니다.

server$ cd /scratch/$USER/qe_launch_test server$ qsub run_qe_test.pbs # 예: 22365267.pbs 같은 JOBID가 출력됩니다.

왜 작은 테스트부터?

  • queue와 module이 맞는지 확인합니다.
  • mpirun, pw.x, xspectra.x 경로를 확인합니다.
  • 큰 계산 전에 script 문법 오류를 빨리 찾습니다.

launch test는 진짜 과학 계산이 아니라 “실행 환경 확인”입니다.

Common mistakesfixes for beginners

자주 하는 실수는 대부분 위치 확인과 파일 확인으로 막을 수 있습니다.

1. home에서 qsub

해결: cd /scratch/$USER/내_계산_폴더 후 제출.

2. server$까지 입력

해결: prompt label 뒤의 명령어만 입력.

3. scratch에만 결과 보관

해결: 계산 후 중요한 파일을 home으로 복사.

4. #PBS -A qe 누락

해결: QE PBS script 상단에 application/account 줄 추가.

큰 계산을 로그인 노드에서 직접 오래 실행하지 마세요. 항상 PBS script로 제출합니다.
