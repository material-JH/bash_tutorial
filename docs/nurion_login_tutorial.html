Nurion login tutorialbefore Linux terminal class

누리온에 먼저 로그인하고, 그 안에서 리눅스 터미널을 배웁니다.

이 자료의 목표는 계산이 아니라 “모든 학생이 같은 서버 화면에 들어오는 것”입니다. 로그인 후에는 pwd, ls, cd부터 시작합니다.

1. 내 컴퓨터

Terminal, PowerShell, Windows Terminal을 엽니다.

2. SSH 로그인

누리온 주소와 사용자 ID로 접속합니다.

3. 서버 확인

접속 후 hostname, pwd로 위치를 확인합니다.

Mental modeltwo computers

지금 명령어가 내 노트북에서 실행되는지, 누리온에서 실행되는지 구분합니다.

내 노트북

→

누리온 로그인 노드

계산 작업 위치는 다음 발표자가 설명합니다.

주의: local$, server$ 글자는 설명용입니다. 실제로 입력하지 않습니다.

Step 1open a terminal

먼저 내 컴퓨터에서 터미널을 엽니다.

Windows

  • Windows Terminal 권장
  • 또는 PowerShell 사용
  • 검색창에서 “Terminal” 입력

macOS

  • Terminal 앱 실행
  • Spotlight에서 “Terminal” 검색
  • 또는 iTerm2 사용 가능

처음 수업에서는 터미널 앱을 여는 것부터 실습입니다. 파일 탐색기가 아니라 글자로 서버에 접속합니다.

Step 2ssh to Nurion

누리온 로그인 명령은 이 형태입니다.

local$ ssh -l 사용자ID nurion.ksc.re.kr

예시

local$ ssh -l a2114a01 nurion.ksc.re.kr

실제 수업에서는 본인 계정 ID를 넣습니다.

비밀번호 입력

  • 입력해도 화면에 글자가 보이지 않을 수 있습니다.
  • 정상입니다. 입력 후 Enter를 누릅니다.
  • 틀리면 다시 천천히 입력합니다.
First connectionfingerprint question

처음 접속하면 fingerprint 질문이 나올 수 있습니다.

The authenticity of host 'nurion.ksc.re.kr (...)' can't be established. Are you sure you want to continue connecting (yes/no/[fingerprint])?

이 질문이 나오면 yes를 입력하고 Enter를 누릅니다.

local$ yes
Real examplePowerShell login screen

Windows PowerShell에서는 실제로 이런 순서로 보일 수 있습니다.

PS C:\Users\USER> ssh a2114a01@nurion.ksc.re.kr (a2114a01@nurion.ksc.re.kr) Password(OTP): (a2114a01@nurion.ksc.re.kr) Password: Last login: Thu May 21 13:48:08 2026 from 220.67.148.1

Password(OTP)

OTP 또는 1차 인증 값을 입력하는 단계입니다. 화면에는 입력한 글자가 보이지 않을 수 있습니다.

Password

계정 비밀번호를 입력합니다. 틀리면 천천히 다시 입력합니다.

Last login

로그인 성공 후 이전 접속 시간이 보이면 서버에 들어온 것입니다.

Login bannerdo not read everything now

로그인 후 긴 안내문이 나오면, 지금은 세 가지만 보면 됩니다.

1. 당황하지 않기

KISTI/NURION 공지, queue 정책, 계정 정보가 한 번에 출력됩니다. 정상 화면입니다.

2. /scratch 경고

/scratch의 오래 사용하지 않은 파일은 삭제될 수 있습니다. 중요한 파일은 보관 위치에 백업합니다.

3. 프롬프트 찾기

공지문이 끝난 뒤 명령어를 입력할 수 있는 server$ 상태가 되었는지 확인합니다.

================ KISTI 5th NURION System ==================== * Queue Policies ... * Mandatory PBS Application Name option (#PBS -A App_Name) ... +---------------------------------------------------------------+ |[WARNING] Files in /scratch unused for 15 days will be deleted | +---------------------------------------------------------------+ =================== Account Information ==================== * a2114a01 Lustre Filesystem Quota Status ...
Step 3confirm where you are

로그인에 성공하면 바로 세 가지를 확인합니다.

server$ hostname server$ whoami server$ pwd

hostname

서버 이름 확인. 예: login01

whoami

내 계정 ID 확인.

pwd

현재 폴더 위치 확인.

Expected resultwhat success looks like

성공하면 대략 이런 결과를 봅니다.

server$ hostname login01 server$ whoami 내_사용자ID server$ pwd /home01/내_사용자ID

여기까지 되면 리눅스 터미널 수업을 시작할 준비가 끝났습니다.

Prompt ruledo not type server$

수업 자료의 server$는 “서버에서 입력”이라는 표시입니다.

맞는 입력

server$ pwd

실제로는 pwd만 입력합니다.

틀린 입력

server$ pwd

server$까지 복사하면 명령어가 아닙니다.

Previewhome and scratch

나중에 큰 작업을 할 때는 home과 scratch를 구분합니다.

/home01/$USER

  • 로그인 후 처음 보이는 내 공간
  • 중요 파일 보관
  • 입력 파일과 결과 백업

/scratch/$USER

  • 큰 작업을 위한 임시 공간
  • 나중에 정리될 수 있음
  • 자세한 사용법은 homework에서 읽기

오늘 첫 단계는 로그인입니다. 파일 이동은 리눅스 기본 명령을 배운 뒤 다시 다룹니다.

File transferMobaXterm + data mover node

파일 전송은 MobaXterm에서 nurion-dm.ksc.re.kr로 접속합니다.

초보자용 기억법

  • nurion.ksc.re.kr: 명령어를 입력하는 SSH 로그인
  • nurion-dm.ksc.re.kr: 파일을 올리고 내려받는 SFTP 접속
  • MobaXterm의 파일 창에서 drag & drop으로 전송합니다.
Remote host: nurion-dm.ksc.re.kr Username: 본인 누리온 ID Protocol: SFTP Remote folder: /home01/본인ID

로그인 노드에서는 SFTP/FTP를 쓰지 않고, 파일 전송용 DM node를 사용합니다.

Troubleshootinglogin problems

로그인이 안 될 때는 에러 메시지를 그대로 봅니다.

Permission denied

ID 또는 비밀번호가 틀렸을 가능성이 큽니다.

Could not resolve hostname

주소 오타 또는 네트워크 문제를 확인합니다.

Connection timed out

네트워크/VPN/방화벽 문제일 수 있습니다.

질문할 때는 마지막 에러 메시지 한 줄을 그대로 보여 주세요.

Logoutend the session safely

수업이 끝나면 exit로 누리온 접속을 종료합니다.

server$ exit Connection to nurion.ksc.re.kr closed. local$

프롬프트가 다시 내 컴퓨터 쪽으로 돌아오면 로그아웃된 것입니다.
